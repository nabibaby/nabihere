1. 在创建项目的时候，经常会出现一些其他的不相关的文件

	或者创建了一些额外的配置，不希望添加到将来的仓库中


2. 例如

	.gitignore     忽略清单的文件

	.idea          编辑器自带的配置文件
	
	.vscode        编辑器自带的配置文件

	node/modules   将来下载的包管理文件夹

	npm-debug.log

	yarn-debug.log

	yarn-error.log   配置文件

	...


	由于某些原因，这些文件都不需要携带进仓库，所以需要忽略

3. 解决办法
	
	创建一个  .gitignore  

	创建方式  使用 Linux 指令或者编辑器的方式创建

	$ touch .gitignore  


	把上述清单文件写入指定忽略文件即可

	
		注释使用 #

		包含关系，所以模糊匹配使用 *



4. 如果文件已经被添加到版本库了，如何取消
	
	$ git rm -r --cached 指定的文件

		删除的是版本库中的文件，把跟踪的引用删了

		保留了工作区的内容

		因为产生了一次删除的记录需要提交，所以不能直接提交
		
		把需要忽略的文件先拿走，再提交，再拿进来设置忽略

	$ git rm -r 指定的文件

		删除的是版本库和工作区的文件
	
		没有办法恢复了
