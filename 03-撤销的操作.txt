
1. 对比工作区和暂存区的代码

	$ git diff

	$ git diff 指定的文件

	要求：没有被add过的文件

2. 对比暂存区和版本库中的代码

	$ git diff --cached 

	$ git diff --cached 指定的文件

	要求：没有被 commit 提交过的文件


	关注的内容: 红色的 - 代表删除的内容

		    绿色的 + 代表新增加的内容


3. 从暂存区撤销add添加，恢复到工作区

	$ git restore --staged ./

	$ git restore --staged 指定的文件

	
	把提交的绿色状态，转换为红色的状态


4. 从版本库恢复文件到工作区

	$ git restore ./

	$ git restore 指定的文件


	如果还有没有add和commit的文件，会直接覆盖

	新建的文件依然会保留
